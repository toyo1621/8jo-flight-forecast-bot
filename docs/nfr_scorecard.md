# 非機能・品質スコアカード

点数は現時点のコードと運用を10点満点で評価した目安です。実運用の成功率や事故履歴が十分にない項目は、控えめに評価します。

| 観点 | 現在 | 根拠 | 主な残存課題 |
| --- | ---: | --- | --- |
| 機能充足 | 8.0 | 11日表示、3便、複数モデル、台風影響度、詳細履歴 | 外部API仕様変更への契約テストがない |
| 計算妥当性 | 4.0 | 便別母集団、未知値除外、補正式を明文化 | 校正、時系列外部検証、Brier scoreが未実施 |
| データ品質 | 7.5 | 3便一括検証、fail-closed収集、BigQuery品質検査 | 過去データの欠測と出典精査が残る |
| 可用性 | 7.5 | 静的配信、6時間更新、7時間キャッシュ | 外部通知と稼働率計測がない |
| セキュリティ | 7.5 | WIF、最小workflow権限、pip-audit、CodeQL、Dependabot | ActionsのSHA固定と定期的な権限監査が必要 |
| プライバシー | 8.0 | 個人情報を扱わず、鍵を保存しない | 将来データ項目追加時の再評価が必要 |
| 法務・データガバナンス | 5.0 | SQLダンプ削除、出典・免責表示 | CSVが残存し、ODPT由来履歴の公開範囲と再配布条件は専門確認が必要 |
| 保守性 | 7.5 | 共通便設定、ステータス正規化、BigQuery単一運用、文書 | `web_app.py`が大きく、API取得と組み立ての分割余地がある |
| テスト容易性 | 7.5 | 外部APIをモックしたpytest、失敗系テスト、CI | BigQuery MERGEの統合テストと画面回帰テストがない |
| 監視性 | 6.5 | Actions summary、品質レポート、badge | SLA、メトリクス、外部通知がない |
| 性能・コスト | 8.5 | GitHub Pages、生成時のみAPI/BigQuery利用 | 履歴増加時のクエリ量監視がない |
| ユーザビリティ | 7.5 | モバイル対応、凡例、詳細、取得時刻、免責 | アクセシビリティ自動検査と利用者テストがない |

単純平均は約7.2です。ただし、計算妥当性と法務は他項目で相殺できない独立した重要リスクです。

## 優先順位

1. ODPT利用条件について、CSVと公開ページ上の個別履歴を含めて専門的に確認する
2. 時系列で訓練・検証期間を分け、便別のBrier scoreと信頼度曲線を作る
3. CSV移行完了後に`user_raw_data.csv`をGit履歴も含めて削除する
4. GitHub Actions失敗の外部通知を追加する
5. BigQueryを使ったMERGE・品質検査の統合テストを追加する
6. 主要画面のアクセシビリティとスクリーンショット回帰テストを追加する
