# 八丈島便 運航統計参考値・表示ロジック仕様書

## 1. 値の位置づけ

画面に表示する%は、対象便と同じ便の過去実績に、予報気象とルールベースの補正を適用した**未校正の運航統計参考値**です。Brier score、信頼度曲線、時系列外部検証は未実施であり、実際の運航確率や航空会社の判断を示しません。

内部APIの`probability`という名称は互換性のため維持しますが、利用者向けには「統計参考値」と表示します。

## 2. 母集団と基礎値

対象便ごとに、BigQuery上の同じ便だけを母集団にします。

1. 風向差30度以内、風速差3 m/s以内を検索します。
2. 5件未満なら風向差45度以内、風速差5 m/s以内へ広げます。
3. それでも5件未満なら同じ便の全履歴を使います。
4. `運航`・`運航(条件付)`は1、`欠航`・`条件付き→引返欠航`は0として平均します。
5. 未取得、空、未対応ステータスは0とせず、保存・集計から除外します。

履歴がない場合は暫定値97を返しますが、データ不足の警告と件数0を表示します。この挙動も統計的根拠を持つ事前確率ではありません。

## 3. 気象補正

| 条件 | 倍率 |
| --- | ---: |
| 視程5 km未満 | 0.60 |
| 視程3 km未満 | 0.45 |
| 降水量2 mm/h以上 | 0.85 |
| 降水量8 mm/h以上 | 0.70 |
| 低層雲量80%超 | 0.90 |
| 低層雲量95%以上 | 0.75 |
| 最大瞬間風速15 m/s以上 | 0.90 |
| 最大瞬間風速20 m/s以上 | 0.55 |
| 突風補正に該当せず、平均風速10 m/s以上 | 0.90 |

突風と平均風速の補正は重複適用しません。風向120〜240度かつ平均風速9 m/s以上では「南風注意」を表示しますが、その条件だけでは値を下げません。最終表示の範囲は0〜97です。

これらは運航基準ではなく仮説的なしきい値です。今後、時系列分割した検証データで妥当性を確認する必要があります。

## 4. 台風影響度

[八丈島・東京方面 台風影響目安](https://8jo-typhoon-ensemble.toyo1621.workers.dev/?source=jma)のJMAモードから、日別の`targets.flight.riskLevel`を取得します。

| API値 | 表示 | 倍率 |
| --- | --- | ---: |
| `low` | 補正なし | 1.00 |
| `medium` | 台風接近リスク小 | 0.90 |
| `high` | 台風接近リスク中 | 0.80 |
| `severe` | 台風接近リスク大 | 0.70 |

倍率は同日の全便と、主予報・GFS・ECMWF・JMAの各統計参考値へ適用します。APIは当日を含む11日分を必要とします。日付が欠けている場合は`low`と推測せず、その日は補正なしとして欠測通知を表示します。API全体が失敗した場合は、7時間以内のキャッシュだけを利用できます。

## 5. 天候信頼度

GFS最大31メンバーとECMWF最大31メンバーで統計参考値を再計算し、10パーセンタイルから90パーセンタイルまでの幅を使います。

| 等級 | 中央80%幅 |
| --- | ---: |
| A | 10ポイント以内 |
| B | 20ポイント以内 |
| C | 30ポイント以内 |
| D | 40ポイント以内 |
| E | 40ポイント超 |

これは気象シナリオ間のばらつきであり、予測精度や運航確率の信頼区間ではありません。メンバーが10件未満の場合は予報日までの日数による暫定評価へフォールバックします。

## 6. データ品質と時刻

- 運航結果と気象値はBigQueryだけへ保存します。
- 当日の3便と必須気象値がすべて揃うまで保存しません。
- 入力`NULL`で既存の正常な気象値を上書きしません。
- 同一ステータスの既知欠航理由を`未確認`で上書きしません。
- 便・到着時刻・予報参照時刻は`app_config.FLIGHTS`を唯一の情報源とします。
- 予報参照時刻はANA1891が8時、ANA1893が13時、ANA1895が17時です。
- 表示する更新時刻は、実際に使用している予報バンドルの取得時刻です。
