# 八丈島便 就航確率予測・表示ロジック仕様書 (forecast_spec.md)

八丈島便は気象の影響を極めて受けやすい路線です。本仕様書は、データ蓄積後の「予測エンジン」および「可視化Web UI」を構築するにあたり、八丈島特有の気象特性と航空運航の現実をロジックに落とし込むための仕様指針です。

---

## 1. 霧（視界不良・シーリング）による欠航・引き返しへの対応

### 特有の課題
八丈島空港は山（八丈富士・三原山）に挟まれた地形であり、周囲が海であるため、梅雨時期（5〜7月）や湿った空気が流れ込む際に**濃霧（視界不良）**が頻繁に発生します。これにより、地上風が弱くても「着陸時の滑走路視認不可」による引き返しや欠航が発生します。

### 対応ロジックの設計
* **データ収集**: `data_collector.py` にて Open-Meteo から **`visibility` (視程: km単位)** を収集し始めました。
* **分析閾値 (しきい値) の仮説**:
  * 視程が **5.0 km 以下**、あるいは低層雲量 (`cloud_cover_low`) が **90% 以上**の場合、霧や低い雲（シーリング）による着陸不可リスクが急激に高まります。
  * 過去のデータから「無風に近いのに欠航・引き返しになった便」を抽出し、その時の `visibility` と `cloud_cover_low` の相関関係を数式化します。

---

## 2. 台風接近時の天候急変への対応

### 特有の課題
台風（熱帯低気圧）接近時は、予報数値（風速・風向）がわずかにずれただけで現地の状況が劇的に変化します。予報上の数値が基準値以下であっても、突発的な暴風雨や突風により直前で欠航や引き返しが決まるケースが多いです。

### 対応ロジックの設計
* **急変注意アラートの表示**:
  * 予報データにおいて、突風 (`wind_gusts`) が **15 m/s 以上**、または風速 (`wind_speed`) が **10 m/s 以上**を記録している場合、あるいは気圧の急激な低下がみられる場合を「台風・低気圧影響圏」と判定します。
  * Web UI上では、算出した就航確率のすぐ横に目立つ警告マークと共に以下の文言を表示します。
    > ⚠️ **注意**: 台風（または発達した低気圧）の接近により天候が急変する可能性が極めて高いです。数値上の就航確率に関わらず、ANA公式の最新の運航情報に十分ご注意ください。

---

## 3. 機材繰り（航空会社都合）への対応（100%を出さないルール）

### 特有の課題
就航確率は純粋な「現地の気象」だけで決まるわけではありません。
* 羽田空港での出発遅れや制限
* 使用する航空機の整備不良や別路線での遅延（機材繰り）
* 乗務員のやりくり

など、八丈島がどれほど無風快晴であっても欠航（機材繰り欠航）になる確率が常に数%存在します。

### 対応ロジックの設計
* **上限確率（キャップ値）の適用**:
  * 予測アルゴリズムが気象条件から「就航確率 100%」と算出した場合でも、UI上の最終表示には上限値（キャップ）を設定します。
  * 表示上の最高就航確率は **`95%`** (または `98%`) とし、決して `100%` は表示しません。
* **UI上の補足説明の記載**:
  * 就航確率の表示欄付近に、以下の注記を常時レイアウトします。
    > ※ 本確率は気象条件に基づく統計値です。当日の機材繰りや突発的な機材整備等による欠航リスクがあるため、最高値は 95% と表現しています。

## 🛠️ 将来の予測モデルの構築イメージ

将来的に予測スクリプトを作る際は、上記のルールを反映した以下のようなハイブリッドモデルを実装します。

```python
def calculate_flight_probability(wind_speed, wind_direction, cloud_cover_low, visibility, is_typhoon_warning):
    # 1. ベースとなる風向・風速からの確率算出 (過去統計)
    base_prob = get_wind_historical_probability(wind_speed, wind_direction)
    
    # 2. 霧・雲量による減算補正
    if visibility is not None and visibility < 5.0:
        base_prob *= 0.6  # 視界不良時は確率を大幅引き下げ
    elif cloud_cover_low > 90:
        base_prob *= 0.8  # 低層雲が厚い場合も引き下げ
        
    # 3. 台風警告時の補正 & アラートフラグ
    alert_required = False
    if is_typhoon_warning:
        base_prob *= 0.7
        alert_required = True
        
    # 4. 機材繰り等を考慮した 100% 排除のキャップ処理
    final_prob = min(base_prob, 95.0)  # 最高でも95%にする
    
    return round(final_prob, 1), alert_required
```
